Make sure the default output is headphones

Choose a loudspeaker from the drop-down for AudioContect output

Now:

The two media elements are playing stream received by getUserMedia() to the default output (headphones),

AudioContext is playing a recording to the loudspeaker, so the sound will be picked up by the microphone

It should be heared in your headphones when "Without echo cancellation" media element is playing,

but should not be heard when "With echo cancellation" media element is playing.

Rendering the audio stream from an audio-only getUserMedia() call with an audio element to the default output device.

With echo cancellation: should not pick up AudioContext playback to the loudspeakers

Without echo cancellation: should pick up AudioContext playback to the loudspeakers



Change AudioContext output device:

Switch it to play to the loudspeakers
